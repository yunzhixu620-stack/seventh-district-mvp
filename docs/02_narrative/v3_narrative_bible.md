# 《第七街区：真人测试》V3.0 剧情文本包与 Codex 使用说明

> 用途：本文件是 V3.0 的剧情源文件。请 Codex 不要自行重写剧情，不要即兴扩写关键台词，不要把剧情改回固定谜题。
> 目标：把当前项目从“AI 推理互动剧”改成“都市 RP 社交局”，并用更成熟的悬疑文案、角色动机和局后揭示支撑可玩性。
> 执行原则：Codex 可以调整字段名、文件拆分和 UI 呈现方式，但本文件中的
> 世界观、角色目标、关键事件、台词风格、局内文本和变量真相必须作为
> V3.0 的 narrative source of truth。

---

## 0. 给 Codex 的使用说明

### 0.1 你应该怎么用本文件

请将本文件放入：

```text
docs/02_narrative/v3_narrative_bible.md
```

然后基于本文件同步更新：

```text
src/data/episode001_linxiaEnvelope.ts
src/data/roles.ts
src/data/characters.ts
src/data/locations.ts
src/data/clues.ts
src/data/messages.ts
src/data/endings.ts
src/data/postGameReports.ts
```

如果当前项目中这些文件名不同，请按现有结构映射，但必须保留同等数据粒度。

### 0.2 不允许做什么

1. 不要把本剧情改写成固定凶手、固定谜底的推理游戏。
2. 不要让所有角色都服务“调查者”。
3. 不要把 AI 明牌标注在局内 UI 中。
4. 不要在局内显示“LIVE AI RESPONSE”“AI Generated”等标签。
5. 不要让 LLM 自己生成关键真相、信封含义、角色真实身份、彩蛋 NPC 或结局。
6. 不要把文案写成产品说明、设定说明或流水账。
7. 不要把角色台词写得像客服、面试回答、任务介绍或系统提示。
8. 不要把文本全部压缩成短标签；悬疑氛围要靠具体细节和可疑行为建立。

### 0.3 允许做什么

1. 可以把长文案拆成 UI 适合的短文案，但必须保留原意。
2. 可以为移动端裁剪字数，但需要保留“角色动机、风险、玩法爽点”。
3. 可以把同一段文案拆为 title/subtitle/body。
4. 可以把消息按事件触发拆入不同 data 文件。
5. 可以在保留本文件文本的基础上增加结构化字段。

### 0.4 本文件的设计参考

本文件借鉴的是悬疑游戏的设计方法，不抄袭具体文本。参考方向如下：

- 《Return of the Obra Dinn》：重点借鉴“玩家通过碎片信息重建真相”的结构，而不是依赖长篇解释。
- 《Her Story》：重点借鉴“非线性信息检索”和“玩家通过关键词、矛盾、语义细节追问”的体验。
- 《Disco Elysium》：重点借鉴“角色身份、城市氛围、人物心理和社会关系驱动的侦探 RPG 文案”。
- 《Alan Wake 2》：重点借鉴“悬疑、怪异、日常生活和恐惧交错”的节奏感。

V3.0 的目标不是复制这些游戏，而是把它们的叙事优点转化成“都市 RP + AI 伪真人局”的可执行文本。

## 1. V3.0 剧情总定位

### 1.1 一句话剧情定位

《第七街区：真人测试》V3.0 讲的是一场发生在封闭街区的夜间社交局。
玩家以某个社会身份入场，场上有普通居民、真人位、AI 代打角色、固定
NPC 和系统代理人。林夏失联和一封正在流转的信只是表面事件；真正的
游戏，是判断谁在演你，谁在利用你，谁可能根本不是人。

### 1.2 剧情基调

基调不是“血腥谋杀”，而是“低暴力都市悬疑”。

关键词：

```text
雨夜
便利店白光
旧楼道
公共频道
陌生私信
未读消息
流转的信封
被删除的提醒
过分礼貌的管理员
太像真人的 AI
故意装得像 AI 的人
```

### 1.3 玩家第一感受

玩家进入游戏后，应该立刻感受到：

```text
这不是一个单纯要我破案的游戏。
我选的角色有自己的目的。
别人也在行动。
我不能确定谁可信。
AI 不在台面上，但它可能正在和我演。
```

## 4. 第一章固定框架与变量真相

### 4.1 固定框架

每局固定存在以下要素：

```text
1. 林夏失联或暂时脱离公共频道。
2. 有一个“信封”正在被寻找、保管、伪造或误传。
3. 至少一名角色想公开信封。
4. 至少一名角色想保住信封。
5. 至少一名角色想利用信封换取利益。
6. 管理员会以“降低风险”为由干预公共频道。
7. 场上至少有一个角色的行为不符合其公开身份。
8. 玩家无法在局内确认哪些角色是 AI。
```

### 4.2 变量槽

每局启动时通过 seed 决定以下变量：

```text
envelopeHolder：信封初始持有人
envelopeMeaning：信封真实含义
trueThreat：真正风险源
fakeThreat：最明显但不一定真实的嫌疑点
protectorRole：本局保护者
seekerRole：本局争夺者
systemInterferenceLevel：管理员干预强度
extraNPCEnabled：是否启用额外异常角色
redHerringSet：本局红鲱鱼组合
aiDisguisePattern：AI 角色伪装策略
```

### 4.3 真相版本 A：误会局

```text
版本名：误会局

林夏没有被害，也没有被绑架。
她发现自己被卷入一次“真人测试”的异常样本中，于是主动断联，观察谁会第一时间寻找信封。

信封里不是罪证，而是一张手写名单。
名单上写着几个人的局内行为模式：
“过度主动的人”
“永远中立的人”
“像真人一样犯错的人”
“像 AI 一样正确的人”

本局真正风险不是某个凶手，而是玩家过早相信了“这一定是案件”。
```

### 4.4 真相版本 B：保护局

```text
版本名：保护局

林夏确实留下了信封。
信封里是一段她写给某个参与者的说明，证明她怀疑本局存在“AI 代打位”。

保护者拿到信封后没有公开，是因为一旦公开，管理员会立刻判定该信息为“破坏测试完整性”，并清除相关记录。

本局真正风险是信封被公开得太早。
太早公开不一定带来真相，反而会让信封消失。
```

### 4.5 真相版本 C：交易局

```text
版本名：交易局

信封确实存在，但里面的内容已经被调包。
真正的内容被某个角色拿去作为交换筹码。

这一局没有传统意义上的反派。
每个人都在用自己掌握的信息换取更安全的位置。

林夏失联只是导火索。
真正推动事件的是：有人发现“信封比林夏本人更值钱”。
```

### 4.6 真相版本 D：系统干预局

```text
版本名：系统干预局

信封原本只是普通留言。
但管理员在第一次扫描后，将它标记为“高风险叙事对象”。

从那一刻开始，信封不再只是信封。
它变成了一个测试样本：

谁会抢？
谁会保护？
谁会相信它不存在？
谁会把它拿来证明别人不是人？

本局真正风险是管理员本身。
它不一定邪恶，但它会为了维持测试继续运行而删除真相。
```

### 5.1 外来调查者

#### 外来调查者：角色选择页文案

```text
外来调查者

你不是第七街区的人。
这让你更容易看到异常，也更容易被所有人利用。

你想要什么：
确认林夏失联和信封之间的关系。

你害怕什么：
你查到的每一条线索，都是别人故意递给你的。

你能做什么：
追问、跟踪、交叉验证、公开矛盾。

别人为什么需要你：
你能把碎片拼成一个能被所有人讨论的结论。

玩法关键词：
搜证 / 盘问 / 跟踪 / 交叉验证

难度：中
```

#### 外来调查者：开局身份简报

```text
你收到邀请时，备注栏里只有一句话：

“请以外来者身份入场，不要暴露你提前知道林夏的名字。”

你没有朋友。
没有后台权限。
没有任何人必须相信你。

你的优势是，你不属于这里。
你的劣势也是，你不属于这里。
```

#### 外来调查者：个人目标

```text
在结算前确认：
1. 信封是否真实存在；
2. 信封当前最可能被谁控制；
3. 至少识别一名正在误导公共频道的角色。
```

#### 外来调查者：失败风险

```text
如果你只相信最主动帮你的人，你很可能会成为别人完成目标的工具。
```

#### 街区爆料者：角色选择页文案

```text
街区爆料者

你不一定知道真相。
但你知道什么样的信息能让所有人闭嘴，或者一起吵起来。

你想要什么：
在午夜前发布一条足够有价值的爆料。

你害怕什么：

你能做什么：

别人为什么需要你：
你能让一个没人敢说的怀疑变成公共议题。

玩法关键词：
爆料 / 舆论 / 交易 / 搅局

难度：中高
```

#### 街区爆料者：开局身份简报

```text
你有一个本地频道，粉丝不多，但够吵。

第七街区的人讨厌你。
因为你总能把不该被看见的东西放到台面上。

今晚你不需要最早知道真相。
你只需要比别人更早知道：谁最怕真相被看见。
```

#### 街区爆料者：个人目标

```text
在结算前至少公开一条改变公共频道风向的信息，并避免自己的可信度彻底崩塌。
```

#### 街区爆料者：失败风险

```text
如果你连续发布未经验证的信息，公共频道会把你当成噪音，而不是信息源。
```

## 6. 场景文案

### 6.1 第七街区


## 7. 关键物：信封

### 7.1 信封基础描述

```text
一个没有署名的牛皮纸信封。

“不要交给最想证明自己清白的人。”
```

### 7.2 信封可能内容 A：名单

```text
里面是一张手写名单。

名单上没有姓名，只有六种描述：

1. 第一个主动帮忙的人
2. 永远要求讲证据的人
3. 不承认自己认识林夏的人
4. 把所有事都说成误会的人
5. 过于像真人的人
6. 过于不像真人的人
```

### 7.3 信封可能内容 B：留言

```text
里面是一张被折过很多次的便签。

林夏写道：

“如果你读到这张纸，说明我暂时安全。
别急着找我。
先看他们会怎么解释我的消失。”
```

### 7.4 信封可能内容 C：空信封

```text
信封是空的。

但里面有明显的压痕。
曾经有一张卡片或照片被放在这里。

最麻烦的不是信封里有什么。
而是现在谁拿走了它。
```

### 7.5 信封可能内容 D：系统编号

```text
信封里只有一张打印纸。

纸上是一串系统编号：

TEST-7D-HUMAN-MIXED-ROLE-SAMPLE

编号下面有一行小字：

“异常样本：行为过度接近真实参与者。”
```

## 12. 行动反馈文案

### 12.1 询问 ask

```text
你没有直接指控对方。
你只是把问题放在桌面上，等对方选择怎么绕开它。
```

### 12.2 试探 probe

```text
你故意说出一半真话。
真正有反应的人，通常听见的是另一半。
```

### 12.3 伪装 disguise

```text
你换了一种身份说话。
这一刻，对方相信的不是你，而是你扮演出来的关系。
```

### 12.4 交易 trade

```text
你把信息当成筹码递出去。
对方接不接，不只取决于信息值多少钱，也取决于你看起来值不值得骗。
```

### 12.5 跟踪 follow

```text
你暂时不再说话。
你想知道的是，对方以为没人看见时会去哪。
```

### 12.6 爆料 reveal

```text
你把一条信息推到公共频道。
从现在开始，它不再只属于你。
```

### 12.7 保护 protect

```text
你没有推进真相。
你选择先保住一个人，或者一件东西。
有些局不是靠更快赢的。
```

### 12.8 搅局 disrupt

```text
你制造了一个新话题。
它未必更接近真相，但足够让原本的真相慢下来。
```

## 14. 结局与局后报告

### 14.1 结局：你公开了真相，但输掉了人

```text
你证明了信封真实存在。

公共频道安静了十几秒。
然后所有人开始解释自己为什么早就知道。

林夏仍然没有出现。
信封内容被截图、转发、质疑、折叠。
你赢得了一个事实。
但它已经不再属于任何人。
```

### 14.2 结局：你保住了林夏，但没人相信你

```text
你没有公开信封。
也没有说出林夏的位置。

结算前，许眠给你发来一条消息：

“她还安全。”

公共频道认为你隐瞒了关键证据。
管理员认为局面已经回到可控范围。
你没有成为英雄。
但有些局，本来就没有英雄位。
```

### 14.3 结局：信封被交易

```text
信封没有消失。
它只是换了主人。

江池在最后一分钟退出公共频道。
阿燃发布了一条没有来源的爆料。
管理员将“信封”关键词降权。

你知道自己被利用了。
更糟的是，对方知道你知道。
```

### 14.4 结局：管理员接管

```text
结算前五分钟，公共频道变得异常干净。

没有争吵。
没有指控。
没有林夏。
没有信封。

每个人都收到了同一条系统提示：

“本局测试已恢复稳定。”

你终于明白：
稳定不是事情解决了。
稳定只是没人能继续说下去。
```

### 14.5 结局：你识破了真人测试

```text
你没有完全解开林夏的事。
但你发现了另一件更重要的事：

场上有一个人不是在隐藏真相。
他是在隐藏自己不是人。

局后报告显示，你标记的三个异常对象中，有两个判断正确。
系统没有祝贺你。
它只显示了一行字：

“下一局将调整拟真程度。”
```
